# 3. 画面遷移とUI仕様 (UI Flow & Layout)

### 3.1 アプリ起動時の挙動

  * **初期画面:** **🔌 Devices Mode** を表示。
      * まず機器接続を行わせるため。
  * **保存先設定:** 初回起動時にデフォルト保存先（例: `D:\Data`）がない場合、設定を促す。(これは `Settings` モードで実装されるべき将来の機能)

### 3.2 メイン画面レイアウト (Main Layout)

アプリケーションは単一のウィンドウで構成され、以下の主要な領域に分割される。

```text
+--------------------------------------------------------------------------------+
| [ HEADER ]                                                                     |
| 🌊 NanoPol Controller v0.1 | [Status] | [Theme] [Rec] [Snapshot]                |
+--------------------------------+-----------------------------------------------+
|                                |                                               |
| [ S ]                          |                                               |
| [ I ]                          |                                               |
| [ D ]                          |      [ MAIN CONTENT AREA ]                    |
| [ E ]                          |                                               |
| [ B ]                          |  (選択されたモードに応じて表示が変化)            |
| [ A ]                          |                                               |
| [ R ]                          |                                               |
|                                |                                               |
+--------------------------------+-----------------------------------------------+
| [ LOG PANEL ]                                                                  |
| [INFO] Application initialized successfully...                                |
+--------------------------------------------------------------------------------+
```

*   **Header (上部):**
    *   アプリケーションのロゴ、タイトル、バージョン情報。
    *   現在のシステムステータスを示すバッジ（例: `System Ready`）。
    *   **アクションボタン:**
        *   ライト/ダークモード切り替え
        *   動画撮影の開始/停止 (`Rec`)
        *   静止画の撮影 (`Snapshot`)

*   **Sidebar (左側):**
    *   縦に並んだアイコンボタンで構成されるナビゲーションバー。
    *   各アイコンがアプリケーションのモードに対応している (`Devices`, `Manual`, `Auto`, `Settings`)。
    *   `Help` ボタンも含まれる。
    *   システムがビジー状態（測定中など）の場合、モード切り替えは無効化される。

*   **Main Content Area (中央):**
    *   選択されているモードに応じたUIが表示される、メインの作業領域。
    *   例えば、`Devices` モードではカメラとステージの接続パネルが表示される。

*   **Log Panel (下部):**
    *   アプリケーションのログメッセージ（情報、警告、エラー）を表示するエリア。
    *   `spec.md` v9.0 で示された「ステータスバー」と「リアルタイムグラフ」の役割を統合・代替する。現在の実装では、グラフ表示はまだない。

### 3.3 各モードの詳細仕様

#### ① 🔌 Devices Mode (接続管理)

  * サイドバーの `Cable` アイコンをクリックすると表示される。
  * **Stage Panel:**
      * Port Select (COMx), Baudrate (9600).
      * `[Connect]` / `[Disconnect]`.
      * Status Indicator (Connected/Error).
  * **Camera Panel:**
      * ID Select.
      * `[Connect]` / `[Disconnect]`.
      * Status Indicator.
  * `DevicesView.tsx` にて実装されている。

#### ② 🛠️ Manual Mode (調整)

  * サイドバーの `HandMetal` アイコンをクリックすると表示される。
  * **Stage Control:**
      * `Current Angle`: 現在の角度を表示。
      * Jog (`[<]`, `[>]`), Absolute Move (`[Go]`, `[Home]`) など、手動でのステージ制御機能。
  * **Camera View:**
      * カメラからのライブ映像を表示。
      * ROI（関心領域）の表示と操作（現在は未実装）。
  * `ManualView.tsx` にて実装されている。

#### ③ 📉 Auto Mode (自動測定)

  * サイドバーの `Activity` アイコンをクリックすると表示される。
  * **現状の実装:**
      * UIはプレースホルダーであり、「▶️ Auto Mode Area」というテキストが表示されるのみ。
      * `spec.md` v9.0で定義された複雑な状態遷移（Session Entry, Selection, Setup & Run）はまだ実装されていない。

#### ④ ⚙️ Settings Mode (設定)

  * サイドバーの `Settings` アイコンをクリックすると表示される。
  * **現状の実装:**
      * UIはプレースホルダーであり、「⚙️ Settings Mode Area」というテキストが表示されるのみ。
      * パス設定やデバイスの内部パラメータ設定機能はまだ実装されていない。
