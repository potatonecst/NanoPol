# 7. バリデーションおよび数値処理仕様 (Validation Rules)

## 7.1 概要

本ドキュメントは、システムにおけるユーザー入力値のバリデーションルール、およびハードウェア制御に関わる数値処理（ステップ解像度、丸め処理）の仕様を定義する。

## 7.2 定数定義と精度

ハードウェア（GSC-01 ステージコントローラ）の仕様および計算精度に基づき、以下の定数を定義する。

| 定数名            | 値       | 説明                                                                               |
| :---------------- | :------- | :--------------------------------------------------------------------------------- |
| `STEP_RESOLUTION` | `0.0025` | 角度制御における最小分解能（ステップ幅）。入力値はこの値の整数倍である必要がある。 |
| `EPSILON`         | `0.001`  | 浮動小数点演算における誤差許容範囲。倍数判定時の比較に使用する。                   |

## 7.3 バリデーションロジック詳細

### ステップ倍数判定 (`isMultipleOfStep`)

入力された数値 `val` が `STEP_RESOLUTION` の整数倍であるかを判定するロジックは以下の通りである。

**アルゴリズム:**
1. `val` が `0` の場合は `true` (適合) とする。
2. `val` を `STEP_RESOLUTION` で除算し、商 `divided` を求める。
3. `divided` を四捨五入し、最も近い整数 `rounded` を求める。
4. `|divided - rounded|` (差分の絶対値) が `EPSILON` 未満であれば `true`、それ以外は `false` とする。

> **Note:** 単純な剰余演算 (`%`) ではなく、除算と四捨五入を用いることで、IEEE 754 浮動小数点数の微小な誤差を吸収する設計とする。

## 7.5 バックエンド/ハードウェア連携時の整合性

フロントエンドとバックエンドで、数値の扱いに以下の役割分担を持たせる。

1.  **フロントエンド (Validation):**
    *   **役割:** 不正な値の入力防止。
    *   **方針:** 自動的な丸め（Snapping）は行わず、ユーザーに明示的に正しい値（0.0025の倍数）を入力させる。これにより、意図しない値への変更を防ぐ。

2.  **バックエンド (Control):**
    *   **役割:** 物理的な駆動パルスへの変換。
    *   **方針:** `backend/devices/stage_controller.py` 内で `round(Angle * 400)` を使用してパルス数に変換する。
    *   **整合性:** フロントエンドで `0.0025` の倍数であることが保証されていれば、`Angle * 400` は必ず整数（または極めて整数に近い値）となり、`round` による情報の欠落は発生しない。
