# 動的ポート割り当て仕様 (Dynamic Port Allocation)

## 1. 背景と目的
デスクトップアプリ（Tauri）内でバックエンドサーバー（Python/FastAPI）を自動起動する際、固定ポート（例: 8000 や 14201）を使用すると、以下のような原因で「ポート競合（Port in use）」エラーが発生し、バックエンドが起動できず「Offline」状態になる問題があります。

*   **ゾンビプロセス:** 前回のアプリ終了時にバックエンドプロセスが適切にキルされず裏で生き残り、ポートを占有し続けているケース。
*   **起動遅延:** 初回の`.exe`起動時の解凍処理などで時間がかかり、ユーザーがアプリ画面を閉じてしまった直後にバックエンドが立ち上がり、そのままゾンビ化するケース。
*   **他アプリとの競合:** 偶然同じポート番号を他のアプリケーションが使用しているケース。

これを**100%確実に防ぎ、何度アプリを起動しても絶対に競合しない堅牢なアーキテクチャ**を実現するため、「動的ポート割り当て」を実装します。

## 2. システム連携フロー（3ステップ）

この仕組みは、HTTP通信（ネットワーク）が確立する「前」の段階で、OSのプロセス間通信（標準出力パイプ）を通じてポート番号を伝達することで成り立っています。

### ステップ 1: Backend (Python) のポート自動取得と通知
1.  Pythonサーバー起動時、`port=0` を指定してUvicornを起動します。
2.  これにより、OS（Windows/Mac）がその瞬間に完全に空いているランダムなポート番号（例: `54321`）を割り当てます。
3.  サーバーは起動直後に、Tauriへ向けた目印として標準出力に `[PORT] 54321` という形式で特別なログを出力します。

### ステップ 2: Tauri (Rust) によるキャプチャと保持
1.  Tauriは `spawn()` でPythonプロセスを裏側で起動した直後から、専用のパイプを通じてその標準出力（黒い画面に出るはずの文字）を常時監視します。
2.  流れてくる文字列の中に `[PORT]` というキーワードを見つけたら、それに続く数字（ポート番号）を抽出します。
3.  抽出したポート番号を、React（フロントエンド）からいつでも問い合わせて参照できるように、Rust側のメモリ空間に安全に保持（メモ）しておきます。

### ステップ 3: Frontend (React) による動的接続
1.  Reactの画面が起動した際、初期化処理（`useEffect`）の中で Tauri の `invoke` コマンドを呼び出し、Rustに対して「今回のバックエンドのポート番号は何番？」と問い合わせます。
2.  Rustから `54321` という返答を受け取ったら、APIクライアント（`client.ts`）の通信先ベースURLを `http://127.0.0.1:54321` に上書きします。
3.  URLの上書きが完了してから、初めてバックエンドへのヘルスチェック（`/health`）やデバイス通信を開始します。

## 3. 期待される効果（メリット）

*   **完全な独立性:** ゾンビプロセスがいくつ裏で残っていようと、毎回新しい空きポートを使うため全く影響を受けずに正常起動できます。
*   **多重起動の許容:** アプリを2つ同時に立ち上げても、それぞれが別のポートを使うため衝突せず、独立して動作します。
*   **タイムラグ耐性:** Pythonの起動（解凍）に何十秒かかろうと、RustとReactはポート番号が流れてくるまで安全に待機できるため、起動タイミングのズレによる通信エラーを完璧に吸収できます。
