# 4. 機能ロジック詳細 (Logic Specification)

> **Note:** 本ドキュメントに記載されている機能は、現時点では **未実装 (Planned)** です。将来の実装指針として参照してください。

### 4.1 ROI 管理・操作 (Planned)

  * **作成:** `Ctrl + Click`。初期サイズ **33x33 px** (奇数)。
  * **編集:**
      * **移動:** 内部ドラッグ。
      * **リサイズ:** ハンドルドラッグで **中心固定・対称拡大縮小**。
  * **表示:** ROI ID番号、枠、中心十字を表示。
  * **右クリックメニュー:** 「数値入力」「削除」。
  * **Zoom/Pan:** カメラビュー上でホイール拡大、背景ドラッグ移動。
      * **実装詳細:** ドラッグ操作時 (`onMouseDown`) に `e.preventDefault()` を呼び出し、ブラウザ標準の画像ドラッグ挙動を無効化することで、スムーズなPan操作を実現する。

### 4.2 オートセンタリング (反復重心法) (Planned)

`[🔍 Pre-Scan]` 時に実行。

1.  **探索:** 現在のROI中心から `Size * 2` の範囲を切り出し。
2.  **閾値:** `Threshold = MaxPixel * 0.2` (背景ノイズ除去)。
3.  **計算:** 輝度重心 $(X_c, Y_c)$ を算出。
4.  **移動:** 現在の中心との差分を **四捨五入(Int)** し、ROIを移動。
5.  **反復:** ズレがなくなるまで、または最大5回繰り返す。
6.  **複数対応:** 全てのROIに対して個別に実行。

### 4.3 測定フロー (Step & Shoot) (Planned)

1.  **検証:** `Laser Power`, `Fiber Pos` 入力済みか？ プレスキャン済みか？ (未実施ならアラート)。
2.  **ループ (0°〜360°):**
      * **移動:** GSC-01 (`M:`/`G:`) → 停止確認 (`Q:` polling) → **整定 0.5秒**。
      * **撮影:**
          * `get_raw_image()` で `numpy array` (生データ) を取得。
          * **表示 (Preview):** 
              * **リアルタイム監視:** MJPEG ストリーミング (`/camera/video_feed`) を利用。
                  * **Cache Busting (キャッシュ対策):** ストリームURLの末尾にタイムスタンプ (`?t=...`) を付与するのは、**接続開始時（src属性セット時）の1回のみ**とする。
                      * **理由:** 切断後の再接続時に、ブラウザが古い切断状態のキャッシュを表示してしまうのを防ぐため。
                      * **注意:** MJPEGは単一のHTTP接続が継続する仕組みであるため、ストリーミング中（フレーム更新ごと）にURLを書き換える必要はない。
              * **静止画確認:** スナップショット撮影時は、取得した静止画を一時的に表示可能。
              * **停止・エラー時:** ライブビュー停止時や未接続時は、黒画面または「No Signal」ダミー画像を表示。
      * **保存:** 
          * **自動測定時:** 生データを **16-bit TIFF** (または PNG) で劣化なく保存。Settings で指定されたパスへ自動書き出し。
          * **マニュアルスナップショット:** ユーザーが「保存」を明示的に指示した場合、保存ダイアログを表示、またはブラウザのダウンロード機能を利用。
      * **解析:** 生データ (Numpy) を用いて全ROIの `Sum`, `Max` を計算。
      * **更新:** グラフプロット、プログレスバー更新。
3.  **飽和監視:**
      * `Max >= SaturationLimit` (10bit: 1023 or 16bit: 65535) の場合、**一時停止ダイアログ** を表示。
      * 選択肢: `[中断して保存]` `[無視して続行]` `[設定を変えてやり直し]`。
4.  **終了処理:**
      * `settings.json` ステータス更新。
      * **グラフ保存:** 最終グラフを画像として保存。
      * **再解析 (Refine):** 全画像から真の重心を再計算し、CSVを更新。
